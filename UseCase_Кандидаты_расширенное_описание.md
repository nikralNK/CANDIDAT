# Расширенное описание Use Case

## 1. Название системы

`Веб-приложение учета кандидатов для приема на службу и формирования ежемесячного Excel-отчета`

Система предназначена для сбора первичных данных от кандидата, ведения кадровиком этапов изучения кандидата и автоматического формирования отчета в формате `.xlsx` по структуре исходной таблицы.

## 2. Акторы

| Актор | Описание | Основная цель |
|---|---|---|
| Кандидат | Физическое лицо, желающее поступить на службу | Заполнить анкету и отправить заявление |
| Кадровик | Сотрудник кадрового подразделения | Вести карточку кандидата, фиксировать проверки, формировать отчет |
| Администратор | Ответственный за настройку системы | Создавать пользователей, назначать роли, вести справочники и настраивать шаблон отчета |

## 3. Границы системы

Внутри системы выполняются:

- регистрация и авторизация пользователей;
- заполнение анкеты кандидата;
- сохранение черновика анкеты;
- отправка заявления кадровику;
- ведение карточки кандидата;
- фиксация этапов изучения;
- фиксация служебных проверок;
- принятие решения по кандидату;
- формирование Excel-отчета.

Вне системы остаются:

- фактическое прохождение ВВК;
- проведение ППО;
- внешние проверки в ИЦ, ГИАЦ, ФСБ, ГИБДД и других органах;
- издание приказа о приеме.

Система только хранит даты, результаты и реквизиты этих действий.

## 6. Детализация основных сценариев

### UC-02. Заполнить личные данные

| Параметр | Описание |
|---|---|
| Актор | Кандидат |
| Цель | Внести первичные персональные данные |
| Предусловие | Кандидат зарегистрирован или вошел в систему |
| Постусловие | Личные данные сохранены в анкете |

Основной поток:

1. Кандидат открывает форму анкеты.
2. Система отображает поля личных данных.
3. Кандидат вводит ФИО.
4. Кандидат вводит дату рождения.
5. Кандидат вводит контактный телефон и email.
6. Система проверяет обязательные поля.
7. Система сохраняет данные.

Альтернативный поток:

| Условие | Действие системы |
|---|---|
| Не заполнено обязательное поле | Система показывает ошибку и не дает перейти дальше |
| Некорректная дата рождения | Система просит исправить дату |

### UC-03. Указать образование и источник сведений

| Параметр | Описание |
|---|---|
| Актор | Кандидат |
| Цель | Заполнить данные, соответствующие колонкам Excel `уровень и вид образования` и `источник получения сведений` |
| Предусловие | Личные данные кандидата заполнены |
| Постусловие | В анкете сохранены образование и источник информации |

Основной поток:

1. Кандидат выбирает уровень образования.
2. Кандидат указывает вид образования.
3. Кандидат вводит учебное заведение или сведения о документе об образовании.
4. Кандидат выбирает источник сведений о трудоустройстве.
5. Если выбран вариант `другое`, кандидат вводит пояснение.
6. Система сохраняет данные.

### UC-06. Отправить заявление кадровику

| Параметр | Описание |
|---|---|
| Актор | Кандидат |
| Цель | Передать заполненную анкету в работу кадровику |
| Предусловие | Все обязательные разделы анкеты заполнены |
| Постусловие | Анкета получает статус `на оформлении` |

Основной поток:

1. Кандидат открывает заполненную анкету.
2. Система показывает итоговую страницу проверки данных.
3. Кандидат подтверждает отправку.
4. Система проверяет обязательные поля.
5. Система присваивает заявлению дату отправки.
6. Система меняет статус на `на оформлении`.
7. Заявление становится доступным кадровику.

Альтернативный поток:

| Условие | Действие системы |
|---|---|
| Есть незаполненные обязательные поля | Система показывает список ошибок |
| Кандидат передумал отправлять | Система оставляет анкету в статусе черновика |

### UC-10. Создать карточку оформления

| Параметр | Описание |
|---|---|
| Актор | Кадровик |
| Цель | Создать рабочую карточку для ведения кандидата по этапам |
| Предусловие | Кандидат отправил заявление |
| Постусловие | Создана карточка оформления кандидата |

Основной поток:

1. Кадровик открывает список новых заявлений.
2. Кадровик выбирает кандидата.
3. Система открывает анкету кандидата.
4. Кадровик проверяет полноту данных.
5. Кадровик нажимает `Принять в работу`.
6. Система создает карточку оформления.
7. Система устанавливает стадию `на оформлении`.

Альтернативный поток:

| Условие | Действие системы |
|---|---|
| Анкета заполнена неполно | Кадровик возвращает анкету кандидату на доработку |
| Кандидат не подходит на первичном этапе | Кадровик фиксирует отказ и причину |

### UC-12. Вести этапы изучения

| Параметр | Описание |
|---|---|
| Актор | Кадровик |
| Цель | Заполнить даты и результаты ключевых этапов оформления |
| Предусловие | Карточка кандидата создана |
| Постусловие | Этапы изучения отражены в карточке и попадут в Excel-отчет |

Основной поток:

1. Кадровик открывает карточку кандидата.
2. Кадровик указывает дату утверждения задания для изучения кандидата.
3. Кадровик указывает дату направления установок УП/УР.
4. Кадровик указывает дату и результат испытаний.
5. Кадровик указывает дату направления на ВВК.
6. Кадровик указывает дату прохождения и результат ВВК.
7. Кадровик указывает дату и результат ППО.
8. Система сохраняет этапы.

### UC-13. Вести служебные проверки

| Параметр | Описание |
|---|---|
| Актор | Кадровик |
| Цель | Зафиксировать даты направления проверок, соответствующие колонкам Excel |
| Предусловие | Карточка кандидата создана |
| Постусловие | Данные проверок сохранены и доступны для отчета |

Основной поток:

1. Кадровик открывает вкладку `Проверки`.
2. Кадровик указывает дату направления проверки ИЦ.
3. Кадровик указывает дату направления проверки ГИАЦ.
4. Кадровик указывает дату согласования ОРЧ СБ.
5. Кадровик указывает дату направления материалов в ФСБ.
6. Кадровик указывает дату проверки ГИБДД.
7. Кадровик указывает даты проверок БСТМ, СООП, УВМ.
8. Кадровик указывает дату проверки диплома.
9. Кадровик указывает дату направления материалов на допуск к СГТ.
10. Система сохраняет данные.

### UC-14. Принять решение по кандидату

| Параметр | Описание |
|---|---|
| Актор | Кадровик |
| Цель | Зафиксировать итог оформления кандидата |
| Предусловие | По кандидату заполнены необходимые этапы и проверки |
| Постусловие | Статус кандидата обновлен |

Основной поток:

1. Кадровик открывает карточку кандидата.
2. Кадровик выбирает итоговое решение.
3. Если кандидат принят, кадровик указывает реквизиты приказа.
4. Если кандидату отказано, кадровик указывает дату и причину отказа.
5. Если оформление продолжается, кадровик оставляет стадию `на оформлении`.
6. Система сохраняет решение.
7. Система обновляет стадию изучения.

### UC-15. Запустить автоматическое формирование Excel-отчета

| Параметр | Описание |
|---|---|
| Актор | Кадровик |
| Цель | Автоматически получить отчет, который открывается в Excel и соответствует исходному шаблону |
| Предусловие | В системе есть карточки кандидатов за отчетный период |
| Постусловие | Сформирован файл `.xlsx` |

Основной поток:

1. Кадровик открывает раздел `Отчеты`.
2. Кадровик выбирает месяц и год отчета.
3. Кадровик нажимает кнопку `Сформировать отчет`.
4. Система автоматически выбирает кандидатов за указанный период.
5. Система автоматически берет данные из анкеты, карточки оформления, этапов, проверок и итогового решения.
6. Система автоматически заполняет строки Excel по структуре исходного файла.
7. Система проставляет порядковые номера.
8. Система формирует готовый файл `.xlsx`.
9. Кадровик открывает или скачивает уже готовый отчет.

Альтернативный поток:

| Условие | Действие системы |
|---|---|
| За период нет кандидатов | Система формирует пустой отчет с заголовками |
| В карточке не заполнены отдельные проверки | В Excel остаются пустые ячейки или прочерк |
| Ошибка формирования файла | Система показывает сообщение и сохраняет ошибку в журнале |
