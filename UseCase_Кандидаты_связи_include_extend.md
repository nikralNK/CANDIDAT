## Что означает include

`<<include>>` означает обязательное включение одного сценария в другой.

Если основной Use Case выполняется, включенный Use Case тоже выполняется всегда.

## Что означает extend

`<<extend>>` означает дополнительный сценарий по условию.

Он выполняется не всегда, а только если возникает определенная ситуация.

## Связи include для проекта

| Основной Use Case | Связь | Включенный Use Case | Объяснение |
|---|---|---|---|
| Выбрать подразделение и должность | `<<include>>` | Фильтровать должности по категории | После выбора категории система всегда показывает только подходящие должности |
| Отправить анкету кадровику | `<<include>>` | Проверить обязательные поля | Перед отправкой система всегда проверяет заполненность анкеты |
| Принять анкету в работу | `<<include>>` | Создать карточку оформления | Когда кадровик принимает анкету, система обязательно создает карточку оформления |
| Запустить автоформирование Excel-отчета | `<<include>>` | Заполнить Excel-шаблон | При формировании отчета система обязательно заполняет шаблон данными |
| Настроить шаблон отчета | `<<include>>` | Настроить соответствие колонок Excel | Чтобы отчет формировался автоматически, нужно указать, какие поля системы попадают в колонки Excel |
| Настроить связи справочников | `<<include>>` | Добавить / редактировать значение справочника | Связи между категорией, службой и должностью настраиваются на основе значений справочников |

## Связи extend для проекта

| Основной Use Case | Связь | Расширяющий Use Case | Условие |
|---|---|---|---|
| Проверить полноту анкеты | `<<extend>>` | Вернуть анкету на доработку | Выполняется только если анкета заполнена неполно или неверно |
| Принять решение по кандидату | `<<extend>>` | Указать причину отказа | Выполняется только если принято решение `отказ` |
| Принять решение по кандидату | `<<extend>>` | Указать реквизиты приказа | Выполняется только если кандидат `принят на службу` |

## Как объяснить на защите

Можно сказать так:

> Связь `include` используется для обязательных действий системы. Например, перед отправкой анкеты система всегда проверяет обязательные поля, а при формировании отчета всегда заполняет Excel-шаблон.

> Связь `extend` используется для условных действий. Например, анкету возвращают на доработку только при ошибках, причину отказа указывают только при отказе, а реквизиты приказа - только при приеме кандидата на службу.
