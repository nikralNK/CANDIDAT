# Use Case для Visio

## Система

`Система учета кандидатов для приема на службу`

> Обновленная расширенная версия диаграммы находится в файле `UseCase_Кандидаты_расширенный_для_Visio.svg`.
> Она лучше подходит для проекта, потому что разделяет личный кабинет кандидата, рабочее место кадровика, администрирование и формирование Excel-отчета.

## Акторы

| Актор | Описание |
|---|---|
| Кандидат | Пользователь, который заполняет анкету и отправляет заявление |
| Кадровик | Сотрудник, который ведет карточку кандидата и формирует отчет |
| Администратор | Пользователь, который управляет справочниками и учетными записями |

## Варианты использования

| ID | Use Case | Актор | Назначение |
|---|---|---|---|
| UC-01 | Заполнить анкету | Кандидат | Ввод персональных данных, образования и желаемой должности |
| UC-02 | Сохранить черновик | Кандидат | Сохранение анкеты без отправки кадровику |
| UC-03 | Отправить заявление | Кандидат | Передача заполненной анкеты в работу кадровику |
| UC-04 | Просмотреть статус | Кандидат | Просмотр текущей стадии изучения |
| UC-05 | Просмотреть список кандидатов | Кадровик | Работа со списком поступивших заявлений |
| UC-06 | Вести карточку кандидата | Кадровик | Просмотр и редактирование данных кандидата |
| UC-07 | Заполнить проверки и результаты | Кадровик | Ввод дат направлений, результатов ВВК, ППО и служебных проверок |
| UC-08 | Изменить стадию изучения | Кадровик | Установка статуса: на оформлении, принят на службу, отказ |
| UC-09 | Запустить автоматическое формирование Excel-отчета | Кадровик | Кадровик выбирает период, а система сама создает файл `.xlsx` по структуре исходного отчета |
| UC-10 | Управлять справочниками | Администратор | Настройка подразделений, должностей, категорий, источников |
| UC-11 | Управлять пользователями | Администратор | Создание, изменение и блокировка учетных записей |

## Как перенести в Visio

1. Открыть Visio.
2. Выбрать шаблон `UML Use Case`.
3. Создать границу системы с названием `Система учета кандидатов для приема на службу`.
4. Разместить акторов:
   - слева: `Кандидат`;
   - справа сверху: `Кадровик`;
   - справа снизу: `Администратор`.
5. Внутри границы системы разместить варианты использования из таблицы.
6. Соединить акторов с вариантами использования обычной ассоциацией.
7. Include-связи показать пунктирной стрелкой с подписью `<<include>>`.
