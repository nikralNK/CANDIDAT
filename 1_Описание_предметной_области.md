# 1. Описание предметной области

Практика проходит в Информационном центре МВД. Одним из направлений деятельности, которое рассматривается в рамках проекта, является учет кандидатов на службу и подготовка отчетных данных по ним. В процессе работы с кандидатами кадровому сотруднику необходимо фиксировать личные сведения кандидата, данные об образовании, источник получения сведений, выбранную категорию службы, должность, наименование службы, этапы изучения, служебные проверки и итоговое решение.

В предметной области учет кандидатов связан с обработкой большого количества разнородной информации. Данные поступают от кандидата в виде документов и сведений, после чего кадровик переносит их в учетные формы. Далее по кандидату ведется процесс изучения: фиксируются этапы, даты направления, результаты прохождения отдельных мероприятий и сведения о служебных проверках. После завершения работы по кандидату принимается итоговое решение: кандидат остается на оформлении, принимается на службу или получает отказ.

В существующем порядке часть информации может обрабатываться вручную, в том числе с использованием Excel-файлов. Такой подход имеет ряд недостатков. При ручном заполнении отчета повышается вероятность ошибок, дублирования сведений и пропуска обязательных данных. Кроме того, кадровику приходится повторно переносить одни и те же сведения в разные формы, что увеличивает время работы и усложняет контроль актуальности информации.

Для решения этих проблем разрабатывается информационная система учета кандидатов. Система представляет собой desktop-приложение на WPF, работающее с базой данных MS SQL Server. Основная задача приложения заключается в автоматизации ввода, хранения и обработки сведений о кандидатах, а также в автоматическом формировании Excel-отчета по заданному периоду.

В разрабатываемой системе данные кандидата вносит кадровик. Кандидат не работает с приложением самостоятельно, так как по уточненным требованиям все сведения поступают кадровому сотруднику и заносятся им в систему. Такой вариант соответствует реальному порядку работы: кадровик проверяет предоставленные данные, выбирает необходимые значения из справочников и создает карточку кандидата.

В системе используются справочники, которые позволяют уменьшить количество ошибок при вводе. Например, категория службы, должность и наименование службы выбираются из выпадающих списков. После выбора категории службы приложение показывает только подходящие должности, а после выбора должности — связанные с ней службы. Благодаря этому пользователь не вводит служебные данные вручную и не подбирает значения случайно.

Основными объектами предметной области являются:

- кандидат;
- карточка оформления кандидата;
- сведения об образовании;
- источник получения сведений;
- категория службы;
- должность;
- наименование службы;
- этапы изучения кандидата;
- служебные проверки;
- итоговое решение;
- отчет по кандидатам.

Основным пользователем системы является кадровик. Он создает карточку кандидата, заполняет сведения, ведет этапы изучения, заносит результаты служебных проверок и принимает итоговое решение. Также в системе предполагается роль администратора, который может управлять пользователями и справочниками, если такая функция будет расширяться в дальнейшем.

В результате автоматизации кадровик получает возможность хранить сведения о кандидатах в единой базе данных, быстро находить нужную карточку, контролировать заполненность данных и формировать отчет без ручного заполнения Excel-таблицы. Это снижает вероятность ошибок, ускоряет подготовку отчетности и делает процесс учета кандидатов более понятным и структурированным.

Таким образом, предметная область проекта связана с автоматизацией кадрового учета кандидатов на службу в Информационном центре МВД. Разрабатываемое приложение позволяет заменить ручное ведение части данных более удобной информационной системой, в которой сведения вводятся через экранные формы, сохраняются в базе данных и используются для формирования отчетной документации.

## Проблемы существующего процесса

В ходе анализа предметной области были выделены следующие проблемы:

- ручное заполнение Excel-отчета занимает дополнительное время;
- одни и те же сведения могут переноситься несколько раз;
- при ручном вводе возможны ошибки в должностях, службах и категориях;
- сложно контролировать полноту заполнения данных кандидата;
- сведения об этапах изучения и проверках могут храниться разрозненно;
- отсутствует единая структура хранения данных по кандидатам.

## Цель автоматизации

Целью автоматизации является разработка приложения, которое позволит кадровику вести учет кандидатов в единой системе и автоматически формировать отчетные данные.

Для достижения цели необходимо решить следующие задачи:

- разработать структуру данных для хранения сведений о кандидатах;
- создать базу данных в MS SQL Server;
- разработать экранные формы для ввода и просмотра данных;
- реализовать выбор значений из справочников;
- добавить сохранение этапов изучения и служебных проверок;
- реализовать сохранение итогового решения по кандидату;
- реализовать автоматическое формирование Excel-отчета.

## Ожидаемый результат

Ожидаемым результатом является WPF-приложение, подключенное к базе данных MS SQL Server. Приложение должно позволять кадровику создавать карточки кандидатов, заносить необходимые сведения, вести этапы изучения и служебные проверки, принимать итоговое решение и формировать Excel-отчет за выбранный месяц.

Разработанная система может использоваться как учебный прототип информационной системы для автоматизации учета кандидатов и подготовки отчетности.
