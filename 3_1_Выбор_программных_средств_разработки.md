# 3.1 Выбор программных средств разработки

При разработке информационной системы учета кандидатов необходимо выбрать программные средства, которые позволяют реализовать удобный интерфейс для кадровика, обеспечить хранение данных в базе данных и сформировать отчет в формате, пригодном для открытия в Microsoft Excel.

Так как разрабатываемая система предназначена для работы на компьютере сотрудника и не требует размещения в сети Интернет, был выбран вариант desktop-приложения. Такой подход удобен для учебного проекта и подходит для предметной области, где пользователь работает с формами ввода, таблицами и отчетами.

## Выбор языка программирования

В качестве языка программирования был выбран **C#**. Данный язык хорошо подходит для разработки приложений под Windows, имеет понятный синтаксис, поддерживает объектно-ориентированное программирование и тесно связан с платформой .NET.

Преимущества выбора C# для данного проекта:

- удобная разработка desktop-приложений;
- поддержка работы с базами данных;
- наличие готовых классов для работы с файлами, датами, строками и коллекциями;
- хорошая интеграция с Visual Studio;
- возможность разрабатывать интерфейс и логику приложения в одном проекте.

В проекте C# используется для обработки действий пользователя, проверки введенных данных, подключения к базе данных, сохранения карточек кандидатов, ведения этапов изучения, служебных проверок и формирования Excel-отчета.

## Выбор технологии разработки интерфейса

Для разработки пользовательского интерфейса была выбрана технология **WPF**. WPF позволяет создавать desktop-приложения с формами, вкладками, таблицами, кнопками, выпадающими списками и другими элементами интерфейса.

Для разрабатываемой системы WPF подходит по следующим причинам:

- приложение должно запускаться на компьютере кадровика;
- интерфейс состоит из экранных форм и таблиц;
- необходимо использовать поля ввода, календарь, выпадающие списки и кнопки;
- удобно разделять разметку интерфейса и программную логику;
- можно быстро дорабатывать внешний вид приложения.

В проекте файл `MainWindow.xaml` отвечает за внешний вид главного окна, а файл `MainWindow.xaml.cs` содержит обработчики событий и логику работы формы.

## Выбор платформы

Для проекта используется **.NET Framework 4.7.2**. Данная платформа подходит для WPF-приложений и поддерживается Visual Studio. Она позволяет использовать необходимые компоненты для создания desktop-приложения, работы с базой данных и формирования файлов.

Выбор .NET Framework связан с тем, что WPF-проекты такого типа удобно создавать и запускать в Visual Studio на операционной системе Windows.

## Выбор системы управления базами данных

В качестве системы управления базами данных была выбрана **Microsoft SQL Server**. Эта СУБД подходит для хранения структурированных данных и часто используется в информационных системах, работающих с таблицами, связями и справочниками.

Преимущества MS SQL Server для данного проекта:

- поддержка реляционной модели данных;
- возможность создавать таблицы, первичные и внешние ключи;
- поддержка связей между справочниками и основными таблицами;
- удобная работа через SQL Server Management Studio;
- возможность использовать хранимые процедуры и триггеры;
- надежное хранение данных.

В базе данных хранятся сведения о кандидатах, образовании, источниках сведений, категориях службы, должностях, службах, этапах изучения, служебных проверках и итоговых решениях.

## Выбор способа подключения к базе данных

Для подключения приложения к базе данных используется **ADO.NET** и пространство имен `System.Data.SqlClient`. Этот способ позволяет напрямую выполнять SQL-запросы из приложения и получать данные из MS SQL Server.

ADO.NET был выбран по следующим причинам:

- не требует сложной настройки;
- позволяет явно видеть SQL-запросы;
- подходит для учебного проекта;
- удобно объяснять работу с базой данных на защите;
- поддерживает параметризованные запросы.

В проекте работа с базой данных вынесена в отдельный класс `DatabaseService`. Это сделано для того, чтобы не смешивать код интерфейса и SQL-запросы. Класс `DatabaseService` выполняет подключение к базе, загрузку справочников, получение карточек кандидатов и сохранение данных.

## Выбор среды разработки

В качестве среды разработки была выбрана **Microsoft Visual Studio**. Visual Studio удобна для создания WPF-приложений, так как поддерживает работу с XAML-разметкой, C#-кодом, проектами .NET Framework и отладкой приложения.

Преимущества Visual Studio:

- удобное создание WPF-проекта;
- встроенный редактор XAML;
- подсветка синтаксиса C#;
- запуск и отладка приложения;
- работа с файлами конфигурации;
- удобная сборка проекта.

В проекте Visual Studio используется для разработки интерфейса, написания кода, настройки подключения к базе данных и проверки работы приложения.

## Выбор средства контроля версий

Для хранения проекта используется **Git** и удаленный репозиторий **GitHub**. Это позволяет сохранять историю изменений, переносить проект на другой компьютер и иметь резервную копию исходного кода.

В репозиторий были добавлены исходные файлы WPF-приложения, документы по проектированию, диаграммы и материалы для отчета. Для исключения служебных файлов Visual Studio и результатов сборки используется файл `.gitignore`.

## Выбор способа формирования отчета

Для формирования отчета используется самописный метод `BuildExcelXml()`, который создает файл в формате Excel XML с расширением `.xls`. Такой способ выбран потому, что для проекта требуется обычная табличная выгрузка без сложных формул и оформления.

Преимущества выбранного способа:

- не требуется запускать Microsoft Excel из приложения;
- не нужно подключать дополнительные библиотеки;
- файл можно открыть в Excel;
- отчет формируется автоматически;
- код формирования отчета легко объяснить.

Метод `BuildExcelXml()` собирает XML-таблицу, а затем файл сохраняется с помощью `File.WriteAllText()`. Это позволяет автоматизировать подготовку отчета и не заполнять Excel вручную.

## Вывод

Для разработки информационной системы учета кандидатов были выбраны следующие программные средства: язык программирования C#, технология WPF, платформа .NET Framework, СУБД MS SQL Server, способ подключения ADO.NET, среда разработки Visual Studio и система контроля версий GitHub.

Выбранные средства соответствуют задачам проекта, так как позволяют создать desktop-приложение с экранными формами, подключить его к базе данных, реализовать проверку вводимых данных и автоматическое формирование Excel-отчета.
