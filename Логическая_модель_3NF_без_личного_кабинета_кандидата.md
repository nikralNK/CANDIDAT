# Логическая модель 3НФ без личного кабинета кандидата

## 1. Назначение модели

Эта версия логической модели учитывает новую логику проекта: кандидат не регистрируется в системе и не заполняет анкету самостоятельно. Все данные кандидата вносит кадровик на основании предоставленных документов и сведений.

Система предназначена для:

- внесения данных кандидата кадровиком;
- выбора категории, подразделения, службы и должности из справочников;
- ведения этапов изучения и служебных проверок;
- фиксации итогового решения;
- автоматического формирования Excel-отчета.

| Тип этапа изучения | Справочник типов этапов |

## 3. Атрибуты сущностей

### 3.1. Пользователь

| Атрибут | Описание |
|---|---|
| ID пользователя | Уникальный идентификатор пользователя |
| ФИО пользователя | ФИО кадровика или администратора |
| Логин | Логин для входа |
| Пароль | Данные авторизации |
| Роль | Кадровик или администратор |

В этой версии кандидат не является пользователем системы.

### 3.2. Кандидат

| Атрибут | Описание |
|---|---|
| ID кандидата | Уникальный идентификатор кандидата |
| ФИО кандидата | Фамилия, имя, отчество кандидата |
| Дата рождения | Дата рождения кандидата |
| Телефон | Контактный телефон |
| Email | Электронная почта |

ФИО хранится в сущности `Кандидат`, потому что кандидат не имеет учетной записи в таблице `Пользователь`.

### 3.9. Вакансия / доступная должность

Эта сущность нужна для фильтрации выпадающих списков.

| Атрибут | Описание |
|---|---|
| ID вакансии | Уникальный идентификатор связки |
| ID категории | Категория службы |
| ID подразделения | Подразделение |
| ID службы | Служба |
| ID должности | Должность |

Кадровик выбирает вакансию из справочников, а система автоматически получает из нее категорию, подразделение, службу и должность.

### 3.10. Карточка оформления

Карточка оформления является основной рабочей сущностью. Она создается кадровиком после получения данных кандидата.

| Атрибут | Описание |
|---|---|
| ID карточки | Уникальный идентификатор |
| ID кандидата | Кандидат, по которому ведется оформление |
| ID источника сведений | Откуда кандидат узнал о трудоустройстве |
| ID вакансии | Выбранная категория, подразделение, служба и должность |
| ID кадровика | Пользователь, который ведет карточку |
| Дата заявления | Дата заявления кандидата |
| Дата создания карточки | Когда кадровик создал запись |
| Стадия изучения | На оформлении, принят, отказ |
| Иные сведения | Дополнительные сведения |

В 3НФ карточка не хранит текстом категорию, подразделение, службу и должность. Она хранит только `ID вакансии`.

## 4. Связи и кардинальности

| Связь | Кардинальность | Объяснение |
|---|---|---|
| Кандидат - Образование | 1 : 0..N | У кандидата может быть несколько записей об образовании |
| Кандидат - Карточка оформления | 1 : 0..N | По кандидату может быть одна или несколько карточек оформления |
| Пользователь - Карточка оформления | 1 : 0..N | Один кадровик может вести много карточек |
| Источник сведений - Карточка оформления | 1 : 0..N | Один источник может быть указан во многих карточках |
| Категория службы - Вакансия | 1 : 0..N | Одна категория используется во многих вакансиях |
| Подразделение - Вакансия | 1 : 0..N | Одно подразделение может иметь много доступных должностей |
| Служба - Вакансия | 1 : 0..N | Одна служба используется во многих вакансиях |
| Должность - Вакансия | 1 : 0..N | Одна должность может встречаться в разных связках |
| Вакансия - Карточка оформления | 1 : 0..N | Одна доступная должность может быть выбрана в разных карточках |
| Карточка оформления - Этап изучения | 1 : 0..N | В карточке может быть много этапов |
| Тип этапа изучения - Этап изучения | 1 : 0..N | Один тип этапа используется во многих записях этапов |
| Карточка оформления - Служебная проверка | 1 : 0..N | В карточке может быть много проверок |
| Тип служебной проверки - Служебная проверка | 1 : 0..N | Один тип проверки используется во многих проверках |
| Карточка оформления - Итоговое решение | 1 : 0..1 | У карточки может быть одно итоговое решение |
| Шаблон отчета - Колонка шаблона отчета | 1 : 1..N | У шаблона есть одна или несколько колонок |
| Шаблон отчета - Выгрузка отчета | 1 : 0..N | По шаблону может быть сформировано много отчетов |
| Пользователь - Выгрузка отчета | 1 : 0..N | Один кадровик может сформировать много отчетов |

## 5. Почему модель соответствует 3НФ

| Правило | Как выполнено |
|---|---|
| Справочные данные не дублируются | Источники, категории, службы, должности, этапы и проверки вынесены в отдельные справочники |
| Карточка не хранит лишний текст | В карточке хранится `ID вакансии`, а не отдельные текстовые значения категории, службы и должности |
| Нет личного кабинета кандидата | Кандидат не связан с пользователем, потому что не входит в систему |
| Повторяющиеся данные вынесены отдельно | Этапы, проверки, образование и колонки отчета находятся в отдельных сущностях |
| Excel формируется из связей | Данные отчета собираются через связанные сущности, а не дублируются в одной таблице |

## 7. Короткое объяснение

> В новой модели кандидат не является пользователем системы. Он предоставляет данные кадровику, а кадровик вносит их в карточку оформления. Для исключения ошибок используются справочники и связка вакансии, которая объединяет категорию, подразделение, службу и должность. Модель приведена к 3НФ, так как справочные значения не дублируются, а основные сущности связаны через идентификаторы.
