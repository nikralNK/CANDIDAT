# Финальная логическая модель данных

## 1. Назначение модели

Логическая модель построена под веб-приложение учета кандидатов. Кандидат заполняет анкету и выбирает значения из справочников, кадровик ведет этапы оформления и проверки, а система автоматически формирует Excel-отчет по шаблону.

Главное изменение после уточнения Use Case: кандидат не вводит должность вручную. Он выбирает категорию, подразделение и должность из выпадающих списков. Система фильтрует доступные должности на основе справочников и связей между ними.

## 2. Основные сущности

| Сущность | Назначение |
|---|---|
| Пользователь | Учетная запись кандидата, кадровика или администратора |
| Кандидат | Персональные данные кандидата |
| Анкета кандидата | Данные, которые кандидат заполняет до отправки кадровику |
| Образование | Сведения об образовании кандидата |
| Источник сведений | Справочник источников: сайт, объявление, рекомендация и т.д. |
| Категория службы | Справочник категорий: юстиция, полиция, внутренняя служба |
| Подразделение | Справочник подразделений / территориальных органов |
| Служба | Справочник служб: СО, УУП, ППС, УР и т.д. |
| Должность | Справочник должностей |
| Связь должности со справочниками | Настраивает, какие должности доступны для категории, службы и подразделения |

| Этап изучения | Задание, УП/УР, испытания, ВВК, ППО |
| Служебная проверка | ИЦ, ГИАЦ, ОРЧ СБ, ФСБ, ГИБДД и другие проверки |
| Итоговое решение | Принят, отказ или на оформлении |
| Шаблон отчета | Правила заполнения Excel-отчета |
| Выгрузка отчета | История автоматически сформированных Excel-файлов |

### 3.10. Связь должности со справочниками

Эта сущность нужна для фильтрации списков. Она показывает, какая должность доступна в какой категории, службе и подразделении.

| Атрибут | Описание |
|---|---|
| ID связи | Уникальный идентификатор |
| ID категории | Категория службы |
| ID службы | Служба |
| ID подразделения | Подразделение |
| ID должности | Должность |
| Активность | Используется ли связка |

Пример:

| Категория | Служба | Подразделение | Должность |
|---|---|---|---|
| Юстиция | СО | ОМВД России по району | Следователь |
| Полиция | УУП | ОМВД России по району | Участковый уполномоченный полиции |
| Полиция | УР | ОМВД России по району | Оперуполномоченный |

## 4. Связи между сущностями

| Связь | Кардинальность | Объяснение |
|---|---|---|
| Пользователь - Кандидат | 1 : 0..1 | Пользователь с ролью кандидата имеет одну анкетную запись, кадровик и администратор ее не имеют |
| Кандидат - Анкета кандидата | 1 : 0..N | Кандидат может создать одну или несколько анкет |
| Кандидат - Образование | 1 : 0..N | У кандидата может быть несколько записей об образовании |
| Источник сведений - Анкета кандидата | 1 : 0..N | Один источник может быть выбран во многих анкетах |
| Категория службы - Связь должности | 1 : 0..N | Одна категория используется во многих допустимых связках |
| Служба - Связь должности | 1 : 0..N | Одна служба используется во многих допустимых связках |
| Подразделение - Связь должности | 1 : 0..N | Одно подразделение может иметь много доступных должностей |
| Должность - Связь должности | 1 : 0..N | Одна должность может быть доступна в разных подразделениях или службах |
| Связь должности - Анкета кандидата | 1 : 0..N | Выбранная связка может использоваться во многих анкетах |
| Анкета кандидата - Карточка оформления | 1 : 0..1 | Карточка появляется только после принятия анкеты в работу |
| Карточка оформления - Этап изучения | 1 : 0..N | В одной карточке много этапов |
| Карточка оформления - Служебная проверка | 1 : 0..N | В одной карточке много проверок |
| Карточка оформления - Итоговое решение | 1 : 0..1 | Решения может еще не быть, но итоговое решение только одно |
| Шаблон отчета - Выгрузка отчета | 1 : 0..N | По одному шаблону можно сформировать много отчетов |
| Пользователь - Выгрузка отчета | 1 : 0..N | Один кадровик может запустить много выгрузок |

## 5. Логика выбора должности кандидатом

1. Кандидат выбирает категорию службы.
2. Система находит активные связи должностей для этой категории.
3. Система показывает только подходящие подразделения и должности.
4. Кандидат выбирает подразделение и должность из списка.
5. В анкете сохраняется не произвольный текст, а ссылка на готовую связку справочников.

Благодаря этому в Excel-отчет автоматически попадают официальные значения:

- категория;
- подразделение;
- служба;
- должность;
- начальствующий состав.

## 7. Вывод

Финальная логическая модель соответствует уточненному Use Case. Она поддерживает выпадающие списки, фильтрацию должностей по категории, работу кадровика с этапами и проверками, а также автоматическое формирование Excel-отчета без ручного заполнения таблицы.
