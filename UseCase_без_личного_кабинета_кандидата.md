# Use Case без личного кабинета кандидата

## 1. Новая логика работы системы

В обновленном варианте кандидат не является пользователем системы и не заполняет анкету самостоятельно.

Процесс выглядит так:

1. Кандидат приносит кадровику свои данные и документы.
2. Кадровик создает запись кандидата в системе.
3. Кадровик вносит личные данные, образование, источник сведений, категорию, подразделение и должность.
4. Система использует справочники и выпадающие списки, чтобы кадровик выбирал корректные значения.
5. Кадровик ведет этапы изучения, служебные проверки и итоговое решение.
6. Система автоматически формирует Excel-отчет по шаблону.

Кандидат в Use Case теперь может быть показан как внешний участник, который предоставляет данные, но он не работает в системе напрямую.

## 2. Акторы

| Актор | Роль |
|---|---|
| Кандидат | Предоставляет кадровику сведения и документы, но не работает в системе |
| Кадровик | Вносит данные кандидата, ведет оформление и запускает автоматическое формирование отчета |
| Администратор | Настраивает пользователей, справочники, связи справочников и шаблон отчета |

## 3. Use Case кандидата

Так как кандидат не входит в систему, у него остается только один внешний сценарий:

| ID | Use Case | Описание |
|---|---|---|
| C-01 | Предоставить сведения и документы | Кандидат передает кадровику ФИО, дату рождения, образование, источник сведений, желаемое подразделение/должность и необходимые документы |

Этот Use Case можно показать на диаграмме как связь кандидата с кадровиком или с системой, но без регистрации и личного кабинета.

## 4. Use Case кадровика

| ID | Use Case | Описание |
|---|---|---|
| H-01 | Создать запись кандидата | Кадровик создает нового кандидата в системе |
| H-02 | Внести личные данные кандидата | Кадровик вводит ФИО, дату рождения и контактные сведения |
| H-03 | Внести сведения об образовании | Кадровик указывает уровень и вид образования кандидата |
| H-04 | Указать источник сведений | Кадровик выбирает, откуда кандидат узнал о возможности трудоустройства |
| H-05 | Выбрать категорию службы | Кадровик выбирает категорию: юстиция, полиция или внутренняя служба |
| H-06 | Выбрать подразделение, службу и должность | Кадровик выбирает данные из выпадающих списков, отфильтрованных по категории |
| H-07 | Создать карточку оформления | Система создает рабочую карточку кандидата для ведения этапов |
| H-08 | Внести этапы изучения | Кадровик заполняет даты и результаты: задание, УП/УР, испытания, ВВК, ППО |
| H-09 | Внести служебные проверки | Кадровик заполняет проверки: ИЦ, ГИАЦ, ФСБ, ГИБДД и другие |
| H-10 | Принять решение по кандидату | Кадровик указывает итог: принят, отказ или на оформлении |
| H-11 | Указать причину отказа | Выполняется только если кандидату отказано |
| H-12 | Указать реквизиты приказа | Выполняется только если кандидат принят на службу |
| H-13 | Запустить автоматическое формирование Excel-отчета | Кадровик выбирает период, а система сама формирует готовый `.xlsx` |

## 5. Use Case администратора

| ID | Use Case | Описание |
|---|---|---|
| A-01 | Создать пользователя | Создание учетной записи кадровика или администратора |
| A-02 | Изменить данные пользователя | Изменение ФИО, логина или роли пользователя |
| A-03 | Добавить значение справочника | Добавление категории, подразделения, службы, должности, источника сведений, типа этапа или проверки |
| A-04 | Редактировать значение справочника | Исправление или уточнение значения справочника |
| A-05 | Настроить связи справочников | Настройка допустимых связок категории, подразделения, службы и должности |
| A-06 | Настроить шаблон Excel-отчета | Настройка структуры отчета и соответствия колонок Excel данным системы |

## 6. Связи include / extend

| Основной Use Case | Связь | Дополнительный Use Case | Объяснение |
|---|---|---|---|
| Внести данные кандидата | `<<include>>` | Проверить обязательные поля | Система проверяет, что ФИО, дата рождения, образование и должность заполнены |
| Выбрать подразделение, службу и должность | `<<include>>` | Фильтровать должности по категории | После выбора категории система показывает только подходящие должности |
| Создать запись кандидата | `<<include>>` | Создать карточку оформления | После внесения данных создается рабочая карточка для дальнейшего оформления |
| Принять решение по кандидату | `<<extend>>` | Указать причину отказа | Выполняется только при отказе |
| Принять решение по кандидату | `<<extend>>` | Указать реквизиты приказа | Выполняется только при приеме |
| Запустить автоматическое формирование Excel-отчета | `<<include>>` | Заполнить Excel-шаблон | Система автоматически заполняет шаблон данными из базы |

## 7. Что убрать из старого Use Case

Из старой версии больше не нужны:

- регистрация кандидата;
- вход кандидата в систему;
- личный кабинет кандидата;
- сохранение анкеты кандидатом;
- отправка анкеты кадровику;
- просмотр статуса кандидатом;
- возврат анкеты кандидату на доработку.

Теперь вся работа с данными кандидата выполняется кадровиком.

## 8. Короткое описание для защиты

> В данной системе кандидат не работает с приложением напрямую. Он предоставляет сведения кадровику, а кадровик вносит данные в систему через рабочее место. Это позволяет централизованно контролировать правильность заполнения данных, использовать справочники и автоматически формировать Excel-отчет без ручного заполнения таблицы.
