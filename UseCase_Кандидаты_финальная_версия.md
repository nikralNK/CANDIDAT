# Финальная версия Use Case

## 1. Общая логика системы

Система автоматизирует работу с кандидатами: кандидат заполняет анкету через веб-приложение, кадровик ведет служебную часть оформления, а Excel-отчет формируется системой автоматически по готовому шаблону.

Важно: кандидат не вводит должность вручную. Он выбирает значения из выпадающих списков. Если кандидат выбирает категорию, система автоматически показывает только подходящие подразделения, службы и должности.

Пример:

1. Кандидат выбирает категорию `юстиция`.
2. Система скрывает должности, которые относятся к полиции или внутренней службе.
3. Кандидат выбирает доступную должность, например `следователь`.
4. Система уже знает связанную службу, категорию и официальное наименование для отчета.

Так кадровику не нужно лишний раз связываться с кандидатом и приводить выбранную должность к официальному виду вручную.

## 2. Акторы

| Актор | Роль в системе |
|---|---|
| Кандидат | Заполняет анкету, выбирает категорию, подразделение и должность, отправляет анкету кадровику, смотрит статус |
| Кадровик | Проверяет анкету, принимает ее в работу, ведет этапы изучения и проверки, принимает решение, запускает автоматическое формирование отчета |
| Администратор | Настраивает пользователей, справочники и правила формирования Excel-отчета |

## 3. Use Case кандидата

| ID | Use Case | Что делает кандидат | Что делает система |
|---|---|---|---|
| C-01 | Зарегистрироваться / войти | Создает учетную запись или входит в систему | Проверяет логин и пароль, открывает личный кабинет |
| C-02 | Заполнить личные данные | Вводит ФИО, дату рождения, контакты | Проверяет обязательные поля |
| C-03 | Указать образование | Выбирает уровень и вид образования, указывает учебное заведение | Сохраняет сведения об образовании |
| C-04 | Указать источник сведений | Выбирает, откуда узнал о работе: объявление, сайт, рекомендация, соцсети и т.д. | Сохраняет источник для Excel-отчета |
| C-05 | Выбрать категорию | Выбирает категорию: юстиция, полиция или внутренняя служба | Фильтрует доступные подразделения, службы и должности |
| C-06 | Выбрать подразделение и должность | Выбирает значения из выпадающих списков | Подставляет официальные связанные данные для отчета |
| C-07 | Сохранить анкету | Сохраняет черновик | Сохраняет данные без отправки кадровику |
| C-08 | Отправить анкету кадровику | Подтверждает отправку | Меняет статус анкеты на `отправлено` |
| C-09 | Просмотреть статус оформления | Открывает личный кабинет | Показывает статус: отправлено, на проверке, на оформлении, принят, отказ |

## 4. Use Case кадровика

| ID | Use Case | Что делает кадровик | Что делает система |
|---|---|---|---|
| H-01 | Просмотреть новые заявления | Открывает список новых анкет | Показывает анкеты со статусом `отправлено` |
| H-02 | Проверить полноту анкеты | Проверяет, все ли обязательные данные заполнены | Подсвечивает пустые или некорректные поля |
| H-03 | Принять анкету в работу | Нажимает `Принять в работу` | Создает карточку оформления и меняет статус на `на оформлении` |
| H-04 | Вернуть анкету на доработку | Указывает, что нужно исправить | Возвращает анкету кандидату и показывает комментарий |
| H-05 | Внести этапы изучения | Заполняет даты и результаты этапов | Сохраняет данные для будущего отчета |
| H-06 | Внести служебные проверки | Заполняет даты направлений и результаты проверок | Сохраняет проверки в карточке кандидата |
| H-07 | Принять решение по кандидату | Выбирает итог: принят, отказ, на оформлении | Обновляет статус кандидата |
| H-08 | Указать причину отказа | Заполняет дату и причину отказа | Сохраняет данные для колонки отказа в Excel |
| H-09 | Указать реквизиты приказа | Заполняет реквизиты приказа при приеме | Сохраняет данные для колонки приказа в Excel |
| H-10 | Запустить автоматическое формирование Excel-отчета | Выбирает период и нажимает кнопку формирования | Автоматически создает готовый `.xlsx` по шаблону |

Кадровик не заполняет Excel вручную. Он только ведет карточки кандидатов и запускает формирование отчета. Система сама берет данные из базы и заполняет Excel-шаблон.

## 5. Use Case администратора

Чтобы диаграмма не была слишком перегруженной, действия администратора лучше показать не одним общим овалом `Управлять справочниками`, а несколькими конкретными действиями.

### 5.1. Пользователи

| ID | Use Case | Что делает администратор |
|---|---|---|
| A-01 | Создать пользователя | Создает учетную запись кадровика или администратора |
| A-02 | Изменить данные пользователя | Меняет ФИО, логин или контактные данные |
| A-03 | Назначить роль пользователя | Назначает роль: кандидат, кадровик, администратор |
| A-04 | Заблокировать пользователя | Отключает доступ к системе |
| A-05 | Сбросить пароль | Выдает пользователю возможность задать новый пароль |

### 5.2. Справочники

Справочники нужны для выпадающих списков. Благодаря им кандидат не гадает, а выбирает только допустимые значения.

| ID | Use Case | Что делает администратор |
|---|---|---|
| A-06 | Добавить значение справочника | Добавляет новую категорию, должность, подразделение, источник сведений или тип проверки |
| A-07 | Редактировать значение справочника | Исправляет название или связанные параметры |
| A-08 | Отключить устаревшее значение | Скрывает значение из выбора, но не удаляет его из старых анкет |
| A-09 | Задать порядок отображения | Настраивает порядок значений в выпадающем списке |
| A-10 | Настроить связи справочников | Связывает категорию, службу, подразделение и должность |

Пример связи справочников:

| Категория | Служба | Доступная должность |
|---|---|---|
| Юстиция | СО | Следователь |
| Полиция | УУП | Участковый уполномоченный полиции |
| Внутренняя служба | Тыл | Специалист |

### 5.3. Шаблон отчета

| ID | Use Case | Что делает администратор |
|---|---|---|
| A-11 | Настроить наименование отчета | Указывает название отчета и подразделение в шапке |
| A-12 | Настроить соответствие колонок Excel | Указывает, из каких полей системы заполняются колонки Excel |
| A-13 | Настроить формат дат | Задает формат отображения дат в отчете |
| A-14 | Настроить правила отбора кандидатов | Указывает, какие статусы и периоды попадают в отчет |

Администратор не заполняет отчет вручную. Он только настраивает правила, по которым система автоматически формирует Excel-файл.

## 7. Нужно ли так подробно показывать администратора на диаграмме

Можно сделать двумя способами.

Вариант для простой диаграммы:

- `Управлять пользователями`;
- `Управлять справочниками`;
- `Настроить шаблон отчета`.

А конкретные действия расписать в описании.

Вариант для расширенной диаграммы:

- `Создать пользователя`;
- `Назначить роль`;
- `Заблокировать пользователя`;
- `Добавить значение справочника`;
- `Редактировать значение справочника`;
- `Отключить устаревшее значение`;
- `Задать порядок отображения`;
- `Настроить связи справочников`;
- `Настроить шаблон отчета`.

Для защиты лучше использовать расширенный вариант, потому что он показывает, что администратор не просто абстрактно "управляет", а выполняет конкретные настройки системы.
